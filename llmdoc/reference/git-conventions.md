# Git 约定

## 1. 核心摘要

本项目采用简单的单分支策略，所有开发直接在 `main` 分支进行，无 `develop` 或功能分支。提交信息使用简洁的中文描述，采用"动作+对象"格式，无严格的 Conventional Commits 规范。

## 2. 源信息

- **主分支:** `main` (唯一的长期分支)
- **远程仓库:** `origin/main`

## 3. 提交信息格式

采用中文描述，格式为：`<动作><对象>`

### 常用动词

| 动词 | 含义 | 示例 |
|------|------|------|
| 优化 | 性能改进或体验提升 | `优化会话列表加载速度` |
| 修复 | Bug 修复 | `修复左下角头像不显示的问题` |
| 增加 | 新增功能 | `增加系统全局提示词` |

### 特点

- 无类型前缀 (如 `feat:`, `fix:`)
- 一行简短描述，不使用多行正文
- 可用 `+` 连接多个修改项：`修复右侧高级菜单太宽的问题+图片生成失效的问题`

## 4. 分支策略

- **单分支模型:** 仅使用 `main` 分支
- **无 PR 流程:** 直接推送到 `main`
- **无功能分支:** 不创建 `feature/*` 或 `bugfix/*` 分支

## 5. 相关参考

- **远程仓库配置:** `.git/config`
