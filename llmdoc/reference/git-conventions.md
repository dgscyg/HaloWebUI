# Git 约定

## 1. 核心摘要

本项目当前并非单分支仓库。长期分支至少包含 `main` 与 `dev`，集成类工作会使用 `merge/*` 分支。提交信息仍以简洁中文为主，采用"动作+对象"格式，无严格的 Conventional Commits 规范。

## 2. 源信息

- **长期分支:** `main`、`dev`
- **集成分支:** `merge/*`，用于显式收敛大规模合并与冲突处理
- **远程命名:** 以当前 `.git/config` 与 `git branch --all` 实况为准

## 3. 提交信息格式

采用中文描述，格式为：`<动作><对象>`

### 常用动词

| 动词 | 含义               | 示例                         |
| ---- | ------------------ | ---------------------------- |
| 优化 | 性能改进或体验提升 | `优化会话列表加载速度`       |
| 修复 | Bug 修复           | `修复左下角头像不显示的问题` |
| 增加 | 新增功能           | `增加系统全局提示词`         |

### 特点

- 无类型前缀 (如 `feat:`, `fix:`)
- 一行简短描述，不使用多行正文
- 可用 `+` 连接多个修改项：`修复右侧高级菜单太宽的问题+图片生成失效的问题`

## 4. 分支策略

- **双长期分支:** `main` 与 `dev` 并存
- **集成分支:** 跨分支合并时可创建 `merge/*` 分支先行收敛冲突
- **提交边界:** 若处于活跃 merge 上下文，提交范围通常按整次 merge 结果处理，而不是任意拆小
